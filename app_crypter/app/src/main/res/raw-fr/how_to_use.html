Comment ça marche ?

Cryptage

  • selectionnez un ficher ou écrire du texte
  • selectionnez un alias de clé ou entrez un mot de passe
  • cliquez sur le boutton Crypter
  • Quand le cryptage est fini, vous pouvez partager le fichier crypté ou le texte crypté

Décryptage

  • selectionnez un ficher crypté ou coller du texte crypté
  • selectionnez un alias de clé ou entrez le mot de passe utilisé pour le cryptage
  • cliquez sur le boutton Dérypter
  • Quand le décryptage est fini, vous pouvez ouvrir le fichier décrypté ou lire le texte décrypté

Keystore

Au lieu d'utiliser un mot de passe à chaque fois, l'utilisateur peut utiliser le Keystore afin de générer des clés lié à un alias. Il suffit alors de sélectionner un alias pour crypter ou décrypter.

Le Keystore est protégé par un mot de passe utilisateur. Toutes les clés générées sont conservées chiffrées sur l'appareil.

Par défault, le mot de passe qui a servi a générer une clé n'est pas conservé.

Si l'utilisateur préfére partager son mot de passe plutôt que la clé générée avec un autre appareils, alors le mot de passe peut être retenu avec un cryptage non réversible. Cela permet à un appareil n'ayant pas la clé de cryptage de déchiffrer un fichier uniquement à l'aide du mot de passe ayant servi à l'encryptage.

Pour retenir un mot de passe (avec cryptage non réversible), l'utilisateur doit explicitement décocher la case Nullifier Password lors de la création de la clé.


Partager une clé

Le partage se fait en 2 étapes avec un protocol ECDH initié par le receveur. Le receveur doit envoyer une clé public à l'émetteur. L'émetteur peut alors crypter sa clé et l'envoyer au réceveur.

  • 1- Receveur ouvrir le Keystore (bouton avec l'icone de clé)
  • 2- Receveur cliquez sur le bouton action(+) (en haut à droite)
  • 3- Receveur cliquez sur le boutton Importer une clé partagée
  • 4- Receveur envoyez la clé public à l'Émetteur (avec une application de messagerie ou affichez le qr code que l'émetteur scannera)

  • 5- Émetteur ouvrir le Keystore (bouton avec l'icone de clé)
  • 6- Émetteur glissez la clé vers la droite et cliquez sur le boutton partager de la clé que vous souhaitez partager
  • 7- Émetteur recevoir la clé public du Receveur (depuis une application de messagerie ou scannez le qr code présenté par le receveur)
  • 8- Émetteur envoyez la clé public au Receveur (avec une application de messagerie ou affichez le qr code que le receveur scannera)

  • 9- Receveur recevoir la clé partagée cryptée de l'Émetteur (depuis une application de messagerie ou scannez le qr code présenté par l'émetteur)
  • 10- Receveur cliquez sur le bouton Confirme pour confirmer l'import de la clé

Stricte mode

Par défault, à chaque encryption le sel, le nombre d'itération, la longueure de clé et l'algorithme de cryptage sont ajoutés aux données cryptées permettant de reconstruire la clé à partir du mot de passe. Cela permet donc de décrypter les données avec le mot de passe ou la clé

L'utilisateur à également la possibilité d'activer l'option Stricte mode. Dans ce mode, aucunes informations permettant de reconstruire la clé à partir du mot de passe n'est ajoutées aux données. Seul la clé permet de décrypter les données (Cela ajoute un niveau de protection contre les attaques par dictionnaires). Avec ce mode, l'utisateur doit faire une sauvegarde de ses clés avec la fonctionnalité Exportez toutes les clés et les conserver dans un endroit sûre.

Perdez ces clés et il sera impossible de décrypter vos données


Notes

-Les fichiers/textes cryptés auront l'extension ".cfx" or ".ctx" et seront dans le dossier android media Documents ou dans le dossier définie par l'utilisateur

-Chaque fichier est encrypté avec une clé AES differente générée aléatoirement à partir d'un mot de passe ou de la clé associé à un alias

-Les fichiers décryptés seront dans l'un des dossiers android media(Pictures, Music, Movies or Documents) ou dans le dossier définie par l'utilisateur

-Les fichiers décryptés reprendront leurs nom d'origines avec peut être un numéro de copie si un fichier avec ce nom existe déjà


Comment le cryptage est fait?

-Le cryptage est fait avec la méthode choisi par l'utilisateur (AES_CTR, AES_CBC, AES_GCM). La méthode AES_GCM est recommendée. Les autres méthodes (AES_CTR, AES_CBC) sont également suffisamment sécuritaire et un peu plus rapide

-Chaque clé secréte est générée à partir d'un mot de passe avec l'algorithme PBKDF2WithHmacSHA1

-La longueure de chaque clé secréte est choisi par l'utilisateur (128, 192, 256). la longueure 256 est recommendée même si (128, 192) sont suffisamment sécuritaire

-Le nombre d'itération et le sel sont générés alératoirement pour chaque nouvelle clé secréte avec l'algorithm SHA1PRNG

-Les données cryptées sont signées avec l'algorithm HmacSHA256

-Pour chaque cryptage un nouveau vecteur d'initialisation généré aléatoirement avec l'algorithm SHA1PRNG est utilisé

-Pour chaque signature un nouveau vecteur d'initialisation généré aléatoirement avec l'algorithm SHA1PRNG est utilisé conjointement avec le tag

-Uniquement pour le chiffrage de texte, l'utilisateur peut choisir un cryptage sans signature ni authentification pour réduire la longueure du message. Cela est suffisement sécuritaire mais nous recommandons de laisser la signature et l'authentification. Si la longueure du message vous pose un souci, il suffit de partager le texte en tant que fichier.

-Le partage de clés est fait avec une paire de clés asymmétriques générées sur chaque appareil et sont echangées avec un protocol ECDH
